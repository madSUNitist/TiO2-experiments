# SEM 粒径自动识别系统 — 开发报告

## 1. 项目概述

本系统实现了从手机拍摄的 SEM（扫描电子显微镜）屏幕照片中自动提取 TiO₂ 颗粒、测量粒径（Feret 直径）、排除团聚体、生成统计分布的全自动化流程。处理对象为 **9 张 SEM 图像**（01-03 为同一批次，06-12 为另一批次），原始图片为手机拍摄电脑屏幕的照片（分辨率 4032×2268 或 5712×3213）。

### 核心技术路线

```
手机照片 → 透视矫正(homography) → 蓝色法裁剪(SEM区域提取)
→ 二值化 → 大团块排除 → 分水岭分割 → 单颗粒分类
→ Feret制图 + 统计 → 合并分布(分段堆叠)
```

## 3. 模块详解

### 3.1 透视矫正（Homography）

**问题**：手机拍摄电脑屏幕，存在透视畸变、梯形失真、旋转。

**技术路线演变**：

| 阶段 | 方法 | 问题 |
|------|------|------|
| ① 梯度法 | 行列亮度梯度找四条边 | 找到的是任意亮度过渡，不是真正的屏幕边框 |
| ② Otsu + HoughLinesP | 阈值分割 + 直线检测四边形 | 内部 UI 线干扰，角点配对错误 |
| ③ 限制区域搜索 | 仅在各边窄带内搜索直线 | 左边框完全检测不到（边缘过渡太柔和） |
| ④ 蓝色边框法（最终） | 颜色分类(BGR 5–50) + 亮度剖面 + 直线合并 | **成功**，详见下文 |

**最终方案 — 基于颜色分类的自适应边框检测**：

- **蓝色掩膜提取**：手机拍摄的屏幕边框和机箱呈现为深色（BGR 5–50，即 RGB `#0B0B0B`–`#323232`），通过三维 BGR 通道范围筛选生成蓝色二值掩膜
- **机箱排除**：底部 1/3 区域检测列密度 > 50% 的大块蓝色区域（机箱），清零
- **Canny + HoughLinesP**：在蓝色掩膜边界上做边缘检测 + 直线检测
- **中心区域排除**：删除触及图像中心 50%×50% 区域的线（排除 SEM 内部图像边缘干扰）
- **方向过滤**：只保留 ±30° 近水平/近垂直线
- **直线合并**：相同斜率 + 相近截距的片段合并延长
- **40% 长度筛选**：只保留跨度 > 40% 图像尺寸的长线
- **最优四边形搜索**：将长线按中点位置分为 top/bottom/left/right 候选，枚举所有 4 线组合，选择**内部蓝色像素数最大化**且面积在 55%–85% 之间的四边形
- **透视变换**：源四边形 → 16:9 矩形（宽度保持源四边形宽度，上限 4000px）

**技术亮点**：
- 放弃了传统基于梯度的边缘检测，转而利用**颜色先验知识**（用户提供了精确的场景颜色信息）构建语义掩膜
- 机箱排除使用 flood-fill + 列密度分析，而非简单的连通域面积阈值
- 最优四边形通过**最大化蓝色掩膜包含量**而非几何特征（如面积、角度）来选择，更鲁棒

**精度验证**：对用户手动标注的三个样本（01-03），右侧边框检测误差仅 1–4 像素（原图 4032px 宽）。

### 3.2 比例尺自动检测（Scale Bar Detection）

**目标**：自动检测 SEM 图像底部的比例尺线段长度，结合已知物理尺寸计算 nm/pixel。

**技术方案**：

- **ROI 裁剪**：根据用户手动标定的 9 个样本平均位置，裁剪区域为图像相对坐标 `x=[0.2545, 0.3251]`、`y=[0.9143, 0.9655]`
- **亮度阈值**：比例尺标记为白色/亮色，阈值 `gray > 190`
- **横线检测**：找到亮像素最密集的行（bar 中心线），再在该行 ±2px 范围内找到连续亮段 → bar 横线范围
- **竖线（tick mark）检测**：在横线范围内逐列检查，是否在 bar 上方和下方均有连续亮像素 → 标记为 tick
- **端点计算**：取最左两 tick 中点和最右两 tick 中点，距离为比例尺像素长度

**物理尺寸推断**：

| 图像组 | 物理尺寸 | bar px 范围 | nm/px 范围 |
|--------|---------|-------------|-----------|
| 01–03 | 3 μm | 164–165 px | 17.8–18.3 |
| 06–12 | 5 μm | 134–162 px | 30.9–37.3 |

两个批次对应不同的 SEM 放大倍率。bar 检测误差 < 2%（与用户手动测量对比）。

---

### 3.3 SEM 图像区域裁剪

**目标**：去除 SEM 软件的蓝色文字标注（如放大倍率、加速电压等），只保留样品图像区域。

**方案**：蓝像素最大空矩形法

- **蓝像素检测**：B > 180 且 B 显著大于 R、G（B > R×1.2, B > G×1.2）→ 标记为 SEM 文字标注
- **边缘裁剪**：先切除图像四周各 5%（去除明显的外围背景）
- **最大空矩形**：在剩余 90% 中心区域中，使用**直方图法**找到内部无蓝色像素的最大轴对齐矩形 — 这是样品的纯净区域

**技术亮点**：
- 利用 SEM 软件文字标注的**特定颜色特征**（蓝色）来定位，比固定坐标裁剪更自适应
- 最大空矩形算法使用经典的**栈式直方图法**，时间复杂度 O(W×H)

---

### 3.4 颗粒分割（Segmentation）

**核心挑战**：
- TiO₂ 颗粒呈不规则块状/多边形
- 亮颗粒（`#CCCCCC`–`#E6E6E6`）分布在深色背景（`#232323`–`#535353`）上
- 大团聚体如果直接送入 watershed 会被错误切碎成许多小颗粒

**流程**：

1. **二值化**：固定阈值 `gray > 170`，颗粒 = 白色
2. **形态学处理**：闭运算连接颗粒碎片 + 膨胀扩大边界（使描边更锐利）
3. **大团块预排除**（关键创新）：
   - `connectedComponents()` 找到所有白色连通域
   - 计算中位面积，排除面积 > 中位面积 × 8 的大连通域
   - **在 watershed 之前执行**，避免大团块被切碎
4. **分水岭分割**：距离变换 → 局部极大值标记 → marker-controlled watershed，分离粘连颗粒
5. **后过滤**：去除面积 < 20px 的噪点、贴边颗粒

**技术亮点**：
- 大团块在 watershed **之前**被移除，从根本上避免了"异常分割导致计数偏差"的问题
- 动态阈值（中位面积 × 8）自适应不同图像的颗粒大小

---

### 3.5 团聚体二次过滤

大团块预排除后，仍可能有少量不规则团块漏过。基于几何特征的二次过滤：

| 特征 | 阈值 | 原理 |
|------|------|------|
| 圆度 | < 0.15 | 团块形状极不规则 |
| 坚固度 | < 0.55 | 团块多孔（凸包面积远大于实际面积） |
| 相对面积 | > 中位 × 8 | 面积异常大 |
| 长宽比 | > 4.0 | 极长形状（碎屑/非颗粒） |

**Feret 直径计算**：
- **Feret max**：凸包顶点对的最大欧氏距离（暴力 O(n²)，对 ≤ 200 点可行）
- **Feret min**：旋转卡尺法，18 个角度扫描找最小投影宽度
- 标注时在颗粒上绘制 Feret 直径线段 + 红字数值

### 3.6 可视化与统计

**标注图**：
- 绿色轮廓 = 单颗粒，红色轮廓 = 被分类为团块
- 蓝色线段 = Feret 直径，红字 = 数值 (nm)
- 白色背景 + 红色文字标注（最终去掉白色背景，直接红字标在 Feret 直径中点）

**粒径分布图** (SVG 格式，透明背景)：
- 蓝色柱状图 (`#2166ac`) + 红色对数正态拟合线 (`#b2182b`)
- 与 XRD 分析图配色一致
- 左上角标注 mean / std / median

**合并分布图**：
- 分段堆叠柱状图，6 组（01-03 均值、06 独立、07-08 均值、09 独立、11 独立、12 独立）
- 黑色对数正态拟合线
- 每组标注颗粒数

## 5. 技术总结

### 关键设计决策

1. **颜色先验驱动**：利用用户提供的精确场景颜色信息（桌面 `#6F6F6F`–`#919191`、边框 `#0B0B0B`–`#323232`、SEM 颗粒 `#CCCCCC`–`#E6E6E6`），构建语义掩膜进行边框检测和区域裁剪，相比纯几何/梯度方法更鲁棒。

2. **分阶段团块排除**：先在 watershed 前移除大联通域（粗过滤），再对分水岭结果做几何特征分类（精过滤），双阶段互补。

3. **自适应参数**：大团块阈值使用中位面积 × 8（而非固定值），使算法对不同放大倍率和颗粒大小的图像自适应。

4. **Feret 直径优于 ECD**：对于不规则块状 TiO₂，最大卡钳直径（Feret）比等效圆直径（ECD）更能反映真实粒径。

5. **可复现性**：所有参数集中在 `config.py` 和硬编码的物理量字典中，全程无随机因素。
